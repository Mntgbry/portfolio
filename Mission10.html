Explication de la table "acteur"

Pour la table acteur, la table IDENT_ACTEUR est la clé primaire. IDENT_ACTEUR apparaît comme une clé étrangère car elle appartient également à la table casting. De type INT

Pour les 4 autres attributs, nom est de type VARCHAR ainsi que prenom. date_naissance est de type DATE, et nb_film de type INT.

Application des contraintes d'intégrité

La contrainte de clé primaire = chaque table à sa clé primaire (PRIMARY KEY).

La contrainte de clé étrangère = chaque clé étrangère devait être une clé primaire. Clés étrangères = id_pays et id_utilisateur pour la table film.

La contrainte de domaine = pour cette contrainte, on a limité les VARCHAR a des caractères (50 la plupart du temps)
